# waldur-cli

A scriptable, LLM/agent-friendly command-line tool for Waldur MasterMind, built on
[rs-client](https://code.opennodecloud.com/waldur/rs-client). Covers two areas of the API,
matching rs-client's own scope:

- **OpenStack** resource management (tenants, instances, volumes, networks, subnets,
  security groups, floating IPs)
- **Team management** (customers, projects, users, roles, invitations, organization groups,
  permission reviews)

This is a curated subset, not full API coverage: only `list`/`get`/`create`/`update`/`delete`
where Waldur exposes a direct REST operation for it. OpenStack tenant/instance/volume
creation and deletion go through Waldur's marketplace ordering flow instead, which isn't
covered here (or by rs-client, which is itself scoped to OpenStack + team-management tags).

`src/commands/` and `src/cli.rs` are generated by
[waldur-cli-generator](https://code.opennodecloud.com/waldur/waldur-cli-generator) — don't
edit them by hand, see that repo for how to regenerate. `src/main.rs`, `src/config.rs`, and
`src/output.rs` are hand-written and permanent.

## Usage

```bash
export WALDUR_API_URL=https://api.example.com
export WALDUR_ACCESS_TOKEN=your-token

waldur-cli openstack tenant list
waldur-cli openstack instance get 00000000000000000000000000000000
waldur-cli team project create --request '{"name": "New project", "customer": "https://api.example.com/api/customers/.../"}'
waldur-cli team customer list --format json   # machine-readable, for scripted/agent use
```

`--api-url`/`--token` flags override the env vars. `create`/`update` commands take the
request body as a `--request` flag containing raw JSON, validated against the same typed
request struct rs-client itself uses — this keeps the CLI simple and avoids needing a flag
per field of every resource's (sometimes large) request schema.
